# 🔍 探索アルゴリズム学習アプリ

高等学校情報Ⅰで学ぶ**線形探索（Linear Search）**と**二分探索（Binary Search）**を体験的に学べる教育的なWebアプリケーションです。

## ✨ 主な機能

### 🎯 2つのアルゴリズムを比較

#### 📝 線形探索（Linear Search）
- **計算量**: O(n) - 線形時間
- **特徴**: 先頭から順番に探索
- **ソート**: 不要
- **最悪ケース**: n回の比較

#### 🎯 二分探索（Binary Search）
- **計算量**: O(log n) - 対数時間
- **特徴**: 毎回探索範囲を半分に削減
- **ソート**: **必須**（昇順または降順）
- **最悪ケース**: log₂n回の比較

### 📊 視覚的な学習機能

#### 線形探索の視覚化
- 🟡 **チェック中**: 黄色でハイライト
- 🔴 **チェック済み**: 赤色（不一致）
- 🟢 **発見**: 緑色でバウンス

#### 二分探索の視覚化
- 🔵 **Left（左端）**: 青色の境界線とLマーク
- 🔵 **Right（右端）**: 青色の境界線とRマーク
- 🟣 **Mid（中央）**: 紫色でハイライト+Mマーク
- ⚪ **範囲外**: グレーアウト表示

### 🎮 インタラクティブ機能

#### 1. アルゴリズム選択
- タブで簡単に切り替え
- 二分探索を選択すると配列が自動的にソート
- ソート済みであることを視覚的に表示

#### 2. 配列サイズ調整
- **スライダー**: 5〜50要素まで自由に調整
- **リアルタイム更新**: スライダーを動かすと即座に配列が再生成
- **レスポンシブ表示**: 配列サイズに応じて自動的にカードサイズを調整

#### 3. ケース別デモ
- 🟢 **最良ケースを試す**: 先頭/中央要素を自動探索
- 🟡 **平均ケースを試す**: 中央要素を自動探索
- 🔴 **最悪ケースを試す**: 存在しない値を自動探索

#### 4. 両アルゴリズム比較
- ワンクリックで両方のアルゴリズムで同じ値を探索
- 比較回数の差を視覚的に表示
- 効率の違いを数値で実感

### 📈 リアルタイム分析

探索実行中に表示される情報：
- **現在のケース判定**: 最良/平均/最悪ケース
- **比較回数**: リアルタイムカウント
- **探索進捗**: パーセンテージとプログレスバー
- **効率**: 配列サイズに対する比較回数の割合
- **状況メッセージ**: 現在の状況を説明

### 📊 詳細な結果分析

探索完了後に表示される情報：
```
📊 詳細分析:
✅ 位置: index 15
✅ 比較回数: 6回
✅ 配列サイズ: 20要素
✅ 探索効率: 30.0%
✅ 計算量: O(log n) - 対数時間
✅ アルゴリズム: 二分探索

💡 学習ポイント:
二分探索では、毎回範囲を半分にするため、たった6回の比較で済みました。
線形探索なら最大20回必要だったところです！
```

### 📚 教育的な解説

#### 計算量の詳細解説
- 最良ケース・平均ケース・最悪ケースの3つを詳しく説明
- 各ケースの発生確率と具体例
- O(1)、O(n)、O(log n)の違いを視覚的に理解

#### なぜ二分探索が速いのか？
- 実用例：辞書で単語を探す方法
- 毎回範囲を半分にする効率性
- 配列サイズと処理時間の関係

#### 配列サイズ別の比較テーブル

| 配列サイズ | 線形探索 O(n) | 二分探索 O(log n) | 効率の差 |
|---------|--------------|-----------------|---------|
| 10要素   | 10回         | 4回             | 2.5倍速い ⚡ |
| 20要素   | 20回         | 5回             | 4倍速い ⚡⚡ |
| 50要素   | 50回         | 6回             | 8.3倍速い ⚡⚡⚡ |
| 100要素  | 100回        | 7回             | 14倍速い 🚀 |
| 1000要素 | 1000回       | 10回            | 100倍速い 🚀🚀 |
| 1,000,000要素 | 1,000,000回 | 20回     | 50,000倍速い 🚀🚀🚀 |

- マウスホバーで詳細情報を確認
- 配列サイズと処理時間の関係を直感的に理解

**グラフ下の情報表示：**


## 🚀 使い方

### 基本的な探索
1. ブラウザで`index.html`を開く
2. アルゴリズムタブで「線形探索」または「二分探索」を選択
3. 探索する値を入力
4. 「🔍 探索開始」ボタンをクリック
5. アニメーションで探索過程を観察

### 配列サイズによる違いを体験
1. 配列サイズスライダーを**5要素**に設定
2. 「🔴 最悪ケースを試す」をクリック
3. 線形探索: **5回**の比較 / 二分探索: **3回**の比較
4. 配列サイズを**20要素**に変更
5. 線形探索: **20回**の比較 / 二分探索: **5回**の比較
6. 配列サイズを**50要素**に変更
7. 線形探索: **50回**の比較 / 二分探索: **6回**の比較

👉 **配列が大きくなるほど、二分探索の優位性が際立つことを実感！**

### アルゴリズム比較デモ
1. 探索する値を入力
2. 「🔬 両アルゴリズムで比較」ボタンをクリック
3. 線形探索と二分探索の結果を並べて表示
4. 比較回数の差を数値で確認

### ステップ実行モード（今後実装予定）
1. 「👣 ステップ実行」ボタンをクリック
2. 1ステップずつ手動で進める
3. 各ステップでの動作を詳しく確認

## 🎓 教育的価値

このアプリを使うことで、以下を**体験的に**学べます：

### 1. 計算量の概念
- **O(1)**: 定数時間 - 配列サイズに関係なく一定

- **O(log n)**: 対数時間 - 配列サイズが2倍でも+1回

### 2. アルゴリズムの効率性
- 配列サイズが大きくなるほど差が顕著に
- 実データでの比較で直感的に理解
- グラフで成長曲線を視覚的に把握

### 3. 前提条件の重要性
- 二分探索は**ソート済み配列が必須**
- アルゴリズムの適用条件を理解
- トレードオフの概念

### 4. 実用的な応用
- 辞書検索 = 二分探索
- データベース検索の最適化
- 実世界での活用例

## 📊 まとめ：線形探索 vs 二分探索

| 項目 | 線形探索 | 二分探索 |
|-----|---------|---------|
| **計算量** | O(n) | O(log n) |
| **ソート** | 不要 ✅ | 必須 ⚠️ |
| **最良ケース** | O(1) - 先頭 | O(1) - 中央 |
| **平均ケース** | O(n/2) | O(log n) |
| **最悪ケース** | O(n) | O(log n) |
| **実装の難しさ** | 簡単 😊 | やや複雑 🤔 |
| **効率性** | 小さい配列では十分 | 大きい配列で圧倒的 🚀 |

**結論**: 配列サイズが大きい場合、二分探索は圧倒的に効率的です。ただし、**ソート済み配列が必須**という前提条件があります！
